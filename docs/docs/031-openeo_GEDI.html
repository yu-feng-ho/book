Use Case: Stream GEDI Data into a Cloud-Processing Workflow¶

- Created by: Johannes Heisig, University of Münster¶

- Date: 28th April 2025¶

Intro¶

This example notebook leverages the cloud-optimized properties of (Geo-)Parquet to stream vector geometries into a server-side cloud computation. The goal is to extract raster cell values from the Copernicus 30-m Digital Elevation Model at GEDI sampling locations and this way compare RADAR- and LiDAR-derived measurements.

gedi_asset = connection.datacube_from_process(
    process_id = "load_url", 
    format = "PARQUET", 
    url = gedi_asset_url)

# DEM data cube 
cop_30 = (connection.load_collection(
    collection_id = "COPERNICUS_30",
    spatial_extent = bbox)
    .reduce_dimension( 
        dimension="t", 
        reducer="mean")
    )
